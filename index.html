توثيق إعداد Samba على Ubuntu

هذا المستند يشرح إعداد مجلد عام يمكن للجميع الدخول إليه بدون كلمة مرور، ومجلد خاص محمي باسم المستخدم osama وكلمة مرور Samba.

ابدأ القراءة

محتويات الصفحة

مقدمة سريعة تجهيز المجلدات ملف الإعدادات الكامل شرح مفصل لملف الإعدادات إضافة المستخدم وتفعيل الخدمة الدخول من ويندوز ملاحظات مهمة
01

مقدمة سريعة

الهدف من هذا الإعداد هو إنشاء مشاركتين على نفس الشبكة المحلية:

المشاركةالاسمنوع الوصولالحمايةالمستخدم
المجلد العامPublicقراءة وكتابةبدون كلمة مرورالجميع
المجلد الخاصPrivateقراءة وكتابةمقيدosama فقط
هذا الدليل مناسب عندما تريد مشاركة ملفات داخل الشبكة المحلية بشكل مرتب، مع فصل واضح بين الملفات المفتوحة للجميع والملفات الخاصة.
02

الأوامر النهائية لتجهيز المجلدات

أول خطوة هي إنشاء المجلدات الفعلية على النظام ثم ضبط الملكية والصلاحيات لكل مجلد حسب الغرض منه.

أوامر الطرفية
# 1. إنشاء المجلدات إذا لم تكن موجودة
sudo mkdir -p /home/osama/public_share
sudo mkdir -p /home/osama/private_share

# 2. ضبط المجلد العام (للجميع)
sudo chown -R nobody:nogroup /home/osama/public_share
sudo chmod -R 777 /home/osama/public_share

# 3. ضبط المجلد الخاص (لأسامة فقط)
sudo chown -R osama:osama /home/osama/private_share
sudo chmod -R 700 /home/osama/private_share
03

ملف الإعدادات الكامل /etc/samba/smb.conf

قبل تعديل الملف، من الأفضل إنشاء نسخة احتياطية:
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak
ملف الإعدادات
# ================= Global Settings =================
[global]
   workgroup = WORKGROUP
   server string = Osama Samba Server
   netbios name = OSAMA-SRV
   security = user
   map to guest = bad user
   dns proxy = no
   socket options = TCP_NODELAY IPTOS_LOWDELAY

# ================= Public Share (No Password) =================
[Public]
   comment = Everyone can access and write here
   path = /home/osama/public_share
   browsable = yes
   writable = yes
   read only = no
   guest ok = yes
   guest only = yes
   force user = nobody
   force group = nogroup
   create mask = 0777
   directory mask = 0777
   force create mode = 0777
   force directory mode = 0777

# ================= Private Share (Password Required) =================
[Private]
   comment = Restricted access for Osama only
   path = /home/osama/private_share
   browsable = yes
   writable = yes
   read only = no
   guest ok = no
   valid users = osama
   create mask = 0700
   directory mask = 0700
04

شرح مفصل لملف الإعدادات (سطر بسطر)

هذا الجدول يوضح وظيفة كل إعداد تم استخدامه في ملف smb.conf.

قسم [global] - الإعدادات العامة

الإعدادالشرح التفصيلي
workgroup = WORKGROUPيحدد اسم مجموعة العمل (Workgroup) التي سيظهر فيها الخادم على الشبكة. يجب أن يكون نفس الاسم المستخدم في أجهزة ويندوز لسهولة الاكتشاف.
server stringوصف نصي للخادم يظهر في تفاصيل الشبكة (مثال: Osama Samba Server).
netbios name = OSAMA-SRVالاسم الفريد للخادم على الشبكة (مثل اسم الكمبيوتر). يستخدم هذا الاسم للوصول المباشر للمجلدات (مثلاً: \\OSAMA-SRV).
security = userيحدد أن الوصول للمجلدات يتطلب مصادقة المستخدمين (اسم مستخدم وكلمة مرور). هذا هو الإعداد الأكثر أمانًا وشيوعًا.
map to guest = bad userعندما يحاول شخص الدخول باسم مستخدم غير موجود في قاعدة بيانات Samba، يقوم هذا الإعداد بمعاملته كـ 'ضيف' (guest). هذا هو المفتاح الذي يسمح بالدخول للمجلد العام بدون كلمة مرور.
dns proxy = noيمنع Samba من محاولة العمل كخادم DNS، وهو إعداد موصى به للأداء والأمان في الشبكات البسيطة.
socket options = TCP_NODELAY...مجموعة من الخيارات لتحسين أداء وسرعة نقل الملفات على الشبكة المحلية.

قسم [Public] - المجلد العام

الإعدادالشرح التفصيلي
commentتعليق وصفي للمجلد يظهر عند استعراض الشبكة.
path = /home/osama/public_shareيحدد المسار الفعلي للمجلد على نظام لينكس.
browsable = yesيجعل المجلد ظاهرًا عند استعراض الشبكة.
writable = yes و read only = noيسمح بالكتابة داخل المجلد. هما وجهان لعملة واحدة.
guest ok = yesيسمح بدخول المستخدمين الضيوف (Guests) بدون كلمة مرور.
guest only = yesيجبر جميع من يدخلون هذا المجلد على أن يكونوا بصلاحيات 'ضيف'، مما يوحد الصلاحيات ويمنع التعارض.
force user = nobodyيجبر أي ملف أو مجلد يتم إنشاؤه على أن يكون ملكاً للمستخدم النظامي `nobody`، وهو مستخدم بدون صلاحيات خاصة، مما يزيد الأمان ويمنع مشاكل الملكية.
force group = nogroupنفس فكرة `force user` ولكن للمجموعات، حيث يجبر الملفات على أن تنتمي للمجموعة `nogroup`.
create mask / directory mask = 0777يضمن أن أي ملف أو مجلد جديد يتم إنشاؤه يحصل على صلاحيات كاملة (قراءة، كتابة، تنفيذ للجميع).
force create mode / force directory modeيفرض الصلاحيات المذكورة في `create mask` حتى لو حاول النظام العميل (مثل ويندوز) تغييرها، وهذا يحل مشاكل الوصول.

قسم [Private] - المجلد الخاص

الإعدادالشرح التفصيلي
path = /home/osama/private_shareيحدد المسار الفعلي للمجلد الخاص على نظام لينكس.
guest ok = noيمنع دخول الضيوف بشكل قاطع. لا يمكن الوصول لهذا المجلد إلا بمستخدم وكلمة مرور صحيحة.
valid users = osamaيحدد قائمة المستخدمين المسموح لهم بالوصول لهذا المجلد. هنا، فقط المستخدم `osama` مسموح له بالدخول.
create mask / directory mask = 0700يضمن أن أي ملف أو مجلد جديد ينشئه `osama` داخل هذا المجلد سيحصل على صلاحيات `700` (قراءة وكتابة وتنفيذ للمالك فقط)، مما يحافظ على خصوصية الملفات.
05

إضافة المستخدم وتفعيل الخدمة

بعد تجهيز المجلدات وتعديل ملف الإعدادات، يجب إضافة المستخدم osama إلى قاعدة بيانات Samba، ثم إعادة تشغيل الخدمة لتطبيق التغييرات.

أوامر نهائية
# 4. إضافة أسامة لقاعدة بيانات السامبا (إذا لم تكن قد فعلت)
sudo smbpasswd -a osama

# 5. فحص الإعدادات للتأكد من عدم وجود خطأ
sudo testparm

# 6. إعادة تشغيل الخدمة لتفعيل كل شيء
sudo systemctl restart smbd
عند تنفيذ أمر smbpasswd -a osama سيطلب منك إدخال كلمة مرور Samba مرتين. هذه الكلمة هي التي ستستخدمها عند الدخول إلى المجلد الخاص من ويندوز.
06

كيف تدخل على المجلدات من ويندوز؟

لتجنب مشكلة تعارض الهويات أو اختلاط تسجيل الدخول بين المجلد العام والمجلد الخاص، استخدم الطريقة التالية بشكل واضح:

الدخول إلى المجلد العام

اكتب هذا المسار مباشرة في شريط العنوان داخل File Explorer:

\\192.168.1.127\Public

هذا المجلد يفتح فوراً بدون طلب اسم مستخدم أو كلمة مرور.

الدخول إلى المجلد الخاص

اكتب هذا المسار مباشرة في شريط العنوان:

\\OSAMA-SRV\Private

سيطلب منك اسم المستخدم وكلمة المرور. أدخل:

Username: osama
Password: Samba Password
السبب في استخدام IP مع المجلد العام واسم الجهاز مع المجلد الخاص هو تقليل فرص حصول تعارض في الجلسات داخل ويندوز.
نصيحة محترف: إذا كنت تريد أن يكون المجلد الخاص ثابتاً دائماً على ويندوز، استخدم Map Network Drive مع تفعيل خيار Connect using different credentials.
07

ملاحظات مهمة قبل التجربة

حتى يعمل كل شيء بشكل صحيح، تأكد من النقاط التالية:

النقطةالتوضيح
تثبيت Sambaتأكد أن الحزمة مثبتة على أوبونتو قبل البدء.
عنوان IPتأكد أن الجهاز يملك IP ثابت أو اعرف عنوانه الحالي.
الجدار الناريإذا كان UFW مفعلاً، اسمح لخدمة Samba بالأمر: sudo ufw allow samba
التحقق من الإعداداتاستخدم sudo testparm للتأكد أن الملف لا يحتوي على خطأ.
إعادة التشغيلأي تعديل في smb.conf يحتاج غالباً إلى sudo systemctl restart smbd
